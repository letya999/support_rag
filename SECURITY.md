# Security Policy

## Reporting a Vulnerability

We take the security of Support RAG seriously. If you believe you have found a security vulnerability, please report it to us responsibly.

**Please do not report security vulnerabilities via public GitHub issues.**

Instead, please send an email to: `security@example.com`

### Our Process

1.  We will acknowledge receipt of your report within 48 hours.
2.  We will provide an estimated timeframe for a fix.
3.  We will notify you once the vulnerability is patched.
4.  We will publicly disclose the vulnerability after a fix has been released, giving you credit for the discovery if you wish.

## Supported Versions

Currently, only the latest version on the `main` branch is supported for security updates.
